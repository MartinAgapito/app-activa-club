# docs/testing

Estrategia de pruebas, casos de prueba y evidencias del proyecto Activa Club
(Vitest, React Testing Library, Playwright).

## Contenido

- [Estrategia de pruebas](./estrategia-de-pruebas.md) — niveles, herramientas,
  criterios de riesgo, política de datos de prueba, responsabilidades por rol,
  cobertura mínima y gestión de defectos ([US-006](../scrum/historias/US-006-estrategia-matriz-pruebas.md)).
- [Matriz de trazabilidad](./matriz-trazabilidad.md) — historia/épica → regla
  (`RN-*`) → caso de prueba → endpoint → tipo de prueba → severidad → estado.

## Estado (Sprint 0)

Ambos documentos son fundación: definen estrategia y casos previstos, sin
implementación de pruebas automatizadas todavía (no hay funcionalidad de
negocio implementada en Sprint 0). Todos los casos de la matriz están en
estado **pendiente**. La implementación de pruebas reales (unitarias,
integración, E2E) comienza en Sprint 1 junto con el desarrollo funcional.
